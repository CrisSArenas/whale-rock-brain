How to use the Whale Rock Brain

Cross-source alt-data synthesis for TMT names. Five minutes of reading, six sources of signal, every claim cited.

Step 1 — Pick a ticker

The dropdown is split into two groups. Case Study Tickers are the five names the Whale Rock team is actively working (AMD, SNDK, FROG, APP, KVYO). Other Examples are larger, broadly-followed names (MDB, NVDA, META, MSFT) included to show the system working across categories.

Step 2 — Pick a window, then click Refresh

The Window dropdown controls how far back each source searches. Default is 1 month. For thin-coverage names (small caps, recent IPOs, niche product companies), bump it to 3 or 6 months for more material. SEC EDGAR always returns the latest filings regardless of window — they're authoritative anchors.

Refresh kicks off the live pipeline: six concurrent source pulls, per-item summarization with Sonnet 4.6, the Brain Summary synthesis, and the cross-source connection judge. A spinner shows the current stage. Cold start is roughly 60-90 seconds; subsequent refreshes of the same ticker are faster because already-summarized items are reused.

If a source column comes back thin, you'll see a "Try N-month window" button on it — one click refreshes with a wider window, no need to fiddle with the dropdown.

Step 3 — Read the metrics strip

Right under the controls, four cards show the last refresh's Duration, LLM Cost, Input Tokens, and Output Tokens. Click the $ button in the header for the per-call breakdown table.

Step 4 — Read the Brain Summary

The navy card at the top is the headline read — what's changing, the non-consensus bear, the non-consensus bull. Every claim cites at least one source item id in square brackets, like R-7f3a. Click any citation chip to scroll the source feed to that exact item with a brief highlight flash.

Step 5 — Scan the source feeds

Six columns, one per source. Each item shows a one-line summary, a sentiment chip (bullish / bearish / neutral / mixed), a relevance score (1-10), and the item id. Click any item to open the original Reddit post, tweet, news article, GitHub issue, or SEC filing in a new tab.

The status row at the top of each column tells you what happened on the last fetch: ok: 23 items means the source returned data; rate-limited or skipped: X_BEARER_TOKEN not set tells you why a column is empty.

Step 6 — Look at the Connections

Below the feeds. Each card is a cross-source link the system surfaced — for example, a Reddit migration thread paired with a GitHub release. Confidence is rated High / Medium by an LLM judge that filters out coincidental keyword overlap. Both ids in the connection are clickable.

Step 7 — Ask the Brain (bottom right)

Click the dock at the bottom-right to expand the chat. It answers questions only from the items currently on screen. If the data doesn't support an answer, it says so plainly — no fabrication. Citations in the answer are clickable, just like in the Brain Summary.

What the system will not do

  • Use general world knowledge to answer questions in chat. If the items don't show it, the chat says so.
  • Surface low-confidence connections. The judge filters them out before they reach the UI.
  • Cite an item id that isn't in the feeds. The frontend cross-checks every citation against the actual items.

Bias notes

Reddit skews retail. GitHub skews developer. X skews short-time-horizon. EDGAR is authoritative but lagged. The Brain flags evidence-thin claims explicitly — treat the dashboard as a starting point, not as a primary source.
